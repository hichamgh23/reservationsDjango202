1    Méthodologie et techniques de développement

Le projet a été développé de manière itérative, fonctionnalité par fonctionnalité, en suivant le plan de cours chapitre par chapitre. Le projet suit le pattern MVT (Model – View – Template) imposé par Django :

  • Model — 13 modèles ORM mappés sur MariaDB (catalogue/models.py)
  • View — fonctions et une classe Python (catalogue/views.py)
  • Template — moteur de templates Django avec héritage via base.html

Principes appliqués : séparation stricte HTML / CSS / JS, DRY via {% extends %} et {% include %}, contrôle d'accès déclaratif par décorateurs.

4    Gestion de la sécurité

Protection CSRF

Le middleware CsrfViewMiddleware est actif sur toutes les requêtes POST. Chaque formulaire inclut le token {% csrf_token %}.

Authentification

  • Système Django natif (django.contrib.auth) — mots de passe hashés PBKDF2-SHA256
  • Sessions sécurisées via django.contrib.sessions
  • Décorateur @login_required sur toutes les vues nécessitant une connexion

Autorisations par profil

ProfilCondition techniqueAccès autorisé
PublicUtilisateur non connectéLecture catalogue, spectacles, artistes, lieux, flux RSS
MEMBERUtilisateur authentifié (groupe MEMBER)Public + réservations + profil + avis + API REST
Superuseris_superuser = TrueTout + CRUD artistes/localités + export CSV + Django Admin

Le décorateur @group_required('ADMIN') défini dans catalogue/views.py autorise les superusers automatiquement, même sans appartenance explicite au groupe ADMIN.

Intégrité de la base de données

Toutes les clés étrangères définissent explicitement ON DELETE et ON UPDATE au niveau SQL (CASCADE, SET NULL ou RESTRICT), en complément du comportement ORM Django.

5    API consommée

DummyJSON Quotes API

ParamètreValeur
NomDummyJSON Quotes API
Adresse (URL)https://dummyjson.com/quotes/random
MéthodeGET
AuthentificationAucune (API publique et gratuite)
Format réponseJSON — champs quote (texte) et author (auteur)
Timeout3 secondes
Gestion d'erreurFail silencieux — la page s'affiche sans citation si l'API est indisponible

Exemple de réponse :

{ "id": 42, "quote": "Life is what happens when you're busy making other plans.", "author": "John Lennon" }

Exploitation dans le projet : Appelée dans la vue welcome() à chaque chargement de la page d'accueil. Les champs quote et author sont transmis au template et affichés dans un encadré stylisé sous l'animation rideau.

# catalogue/views.py — def welcome(request)
response = requests.get('https://dummyjson.com/quotes/random', timeout=3, verify=False)
data     = response.json()
citation = data.get('quote')
auteur   = data.get('author')

9    Procédures de déploiement

La procédure suivante explique comment installer et faire fonctionner le site à partir de l'archive ZIP téléchargée depuis GitHub.

Prérequis — logiciels à installer

  1. Python 3.14+ — https://www.python.org/downloads/ — cocher "Add Python to PATH" lors de l'installation
  2. MariaDB 10.6 — https://mariadb.org/download/ — installer comme service Windows, noter le mot de passe root
  3. HeidiSQL — https://www.heidisql.com/ — client graphique SQL pour administrer MariaDB

Étape 1 — Télécharger et extraire l'archive

  1. Aller sur https://github.com/hichamgh23/reservationsDjango202
  2. Cliquer sur Code → Download ZIP
  3. Extraire l'archive dans le dossier de votre choix (ex : C:\projets\reservationsDjango202\)

Étape 2 — Créer et activer l'environnement virtuel Python

Ouvrir un terminal PowerShell dans le dossier extrait :

python -m venv venv
venv\Scripts\Activate.ps1

Étape 3 — Installer les dépendances Python

pip install -r requirements.txt

Étape 4 — Créer la base de données dans HeidiSQL

  1. Vérifier que MariaDB est démarré (Gestionnaire des tâches → Services → MariaDB)
  2. Ouvrir HeidiSQL → Nouveau → Type : MySQL (TCP/IP) | Hôte : 127.0.0.1 | Port : 3306 | User : root
  3. Dans l'onglet Requête, exécuter :
CREATE DATABASE reservations CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'rooot'@'127.0.0.1' IDENTIFIED BY 'rooot';
GRANT ALL PRIVILEGES ON reservations.* TO 'rooot'@'127.0.0.1';
FLUSH PRIVILEGES;

Étape 5 — Importer le dump SQL

  1. Dans HeidiSQL, sélectionner la base reservations dans le panneau gauche
  2. Menu Outils → Importer un fichier SQL… → sélectionner dump_reservations.sql
  3. Cliquer Ouvrir — tables, contraintes et données créées automatiquement
Le dump contient toutes les tables avec FK (ON DELETE / ON UPDATE) et les données de démonstration complètes.

Étape 6 — Lancer le serveur Django

python manage.py runserver
